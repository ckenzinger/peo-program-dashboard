The "Why": Fixing What's Broken

Our current processes are manual and inconsistent, leading to wasted time and unreliable data. This program will fix these specific, daily frustrations and build a foundation we can trust.

Tackle Inconsistent Data

  • Partner names are messy (e.g., "JPM", "JP Morgan", "JPMC").
  • It's hard to tell if an incident is partner-induced vs. Stripe-caused.
  • We can't easily filter by which partner service is impacted (e.g., acquiring, issuing, treasury).

Reduce Manual Toil

  • PE&O DRIs are not automatically added to relevant incident channels.
  • Incident responders have to manually tag incidents as "partner-induced."
  • Prepping for QBRs requires manually digging through incidents instead of using a dashboard.

Clarify Ambiguous Processes

  • Roles for IRM vs. PE&O vs. Eng are unclear during partner escalations.
  • No clear process for handling long-running "active" incidents waiting on other teams.
  • Unclear when to shut off a payment method during a partner maintenance or outage.

The Path Forward: Our Work Plan

We will execute this program in four distinct phases. Click on each phase below to see the specific action items for that period.

Phase 1: Foundation & Discovery (2 Weeks)

  • Interview PEs and TPaMs about BRB friction points.
  • Analyze historical incident data to quantify impact of poor labeling.
  • Map the current, "as-is" incident response workflow, including all manual steps.

Phase 2: Design & Documentation (4 Weeks)

  • Draft the v2 Runbook with a clear RACI chart for IRM vs. PE&O.
  • Develop the taxonomy for "Partner Functionality" (acquiring, issuing, etc.).
  • Write the formal requirements document for BRB enhancements.
  • Design the V1 Hubble Dashboard mockups.

Phase 3: Implementation & Go-Live (4 Weeks)

  • Submit BRB enhancement requests to the incident reliability tooling team.
  • Conduct training sessions with PE&O on the new runbook and processes.
  • Work with Data Science to build and validate the V1 Hubble dashboard.
  • Announce official "go-live" date for the new program.

Phase 4: Embed & Iterate (Ongoing)

  • Establish and run the monthly PE&O incident review meeting using the new dashboard.
  • Collect feedback via a formal survey 3 months post-launch.
  • Use incident data to create proactive "Partner Improvement Plans" for repeat offenders.
